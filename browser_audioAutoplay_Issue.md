# 브라우저 오디오 자동재생 문제 해결 방법 (Chrome)

웹앱 테스트 중, 앱 첫 화면에서 효과음이나 안내 음성이 자동으로 재생되지 않는 문제가 발생할 수 있다.  
이는 **Chrome의 오디오 자동재생 제한 정책(Autoplay Policy)** 때문이다.

## ✅ 해결 방법: Chrome 사이트 설정을 통해 자동재생 허용

Chrome 설정에서 해당 로컬 주소나 웹사이트를 명시적으로 **"소리 자동재생 허용"** 대상에 추가하면 된다.

---

## 💡 적용 방법 (단계별)

1. **Chrome 주소창에 아래 경로 입력 후 이동**
   `chrome://settings/content/sound`

2. **“소리” 설정 화면에서 아래로 스크롤하여**

- **“소리 재생이 허용됨” 항목**의 `사이트 추가` 버튼 클릭

3. **허용할 사이트 주소 입력**

- 예:
  ```
  http://localhost:3000
  ```
  또는 실제 배포 도메인:
  ```
  https://your-kiosk-domain.com
  ```

위처럼 추가 버튼을 눌러서 자동 재생을 허용할 사이트 주소를 적어주면 끝이다. 4. **사이트를 다시 실행하면 자동재생이 정상 작동됨**

---

## 📝 참고 사항

- 이 설정은 **현재 사용하는 브라우저(Chrome)에만 적용**된다.
- 키오스크 장비 등 다양한 환경에서 동일하게 적용하려면, 각 장비마다 설정을 반복하거나 Kiosk Browser 설정을 따로 구성해야 한다.
- 이 방법외에 SW적으로 해결하는 방법도 있으나, 이 브라우저 내에서 해결하는 방법이 가장 쉽고 편하다.
